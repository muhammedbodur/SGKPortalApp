## 🎯 Önemli Notlar

### 1. Naming Conventions
- **Entity'ler**: Tekil isim (Personel, Departman, Kanal)
- **DTO'lar**: Entity adı + Dto suffix (PersonelDto, DepartmanDto)
- **Repository'ler**: Entity adı + Repository suffix (PersonelRepository)
- **Service'ler**: Entity adı + Service suffix (PersonelService)

### 2. Katmanlar Arası Bağımlılık
```
PresentationLayer → ApiLayer/BusinessLogicLayer
ApiLayer → BusinessLogicLayer
BusinessLogicLayer → DataAccessLayer + BusinessObjectLayer
DataAccessLayer → BusinessObjectLayer
```

### 3. PDKS Modülü Detayları
- **Giriş/Çıkış**: PDKS cihazlarından otomatik veri çekme
- **İzin Yönetimi**: Çok seviyeli onay süreci
- **Mesai Takibi**: Otomatik mesai hesaplama
- **Vardiya Yönetimi**: Esnek vardiya tanımlama
- **Raporlama**: Excel/PDF export desteği

### 4. Eshot Modülü Detayları
- **Hareket Kayıtları**: Tüm kart kullanım geçmişi
- **Kullanım Hakkı**: Personel bazlı hak tanımlama
- **Kart Yönetimi**: Kart tanımlama, aktivasyon, iptal
- **Bakiye Yönetimi**: Otomatik ve manuel bakiye yükleme
- **Ücret Yönetimi**: Tarife tanımlama ve güncelleme

### 5. SignalR Real-time Features
- **Dashboard**: Anlık istatistikler
- **Sıramatik**: Sıra durumu güncellemeleri
- **TV Ekranları**: Canlı sıra gösterimi
- **Bildirimler**: Kullanıcı bazlı bildirimler
- **Personel Durumu**: Online/Offline durumu

### 6. Security Features
- **JWT Token Authentication**
- **Role-based Access Control (RBAC)**
- **Password Hashing** (BCrypt)
- **XSS Prevention**
- **CSRF Protection**
- **SQL Injection Prevention** (Parameterized queries)
- **Rate Limiting**
- **API Key Authentication** (Optional)

### 7. Performance Optimizations
- **Lazy Loading** (Navigation properties)
- **AsNoTracking** (Read-only queries)
- **Pagination** (Tüm listeleme ekranlarında)
- **Caching** (Frequently accessed data)
- **Index Optimization** (Database indexes)
- **Connection Pooling**
- **Response Compression**

## 🚀 Getting Started

### Prerequisites
- .NET 8.0 SDK
- SQL Server 2019+
- Visual Studio 2022 / VS Code
- Node.js (for frontend tooling)

### Installation Steps

1. **Clone Repository**
```bash
git clone https://github.com/your-org/SGKPortalApp.git
cd SGKPortalApp
```

2. **Update Connection String**
```bash
# appsettings.json içinde connection string'i güncelleyin
```

3. **Apply Migrations**
```bash
cd SGKPortalApp.DataAccessLayer
dotnet ef database update
```

4. **Run Seed Data** (Optional)
```bash
# Program.cs içinde seed metodunu aktif edin
```

5. **Run Application**
```bash
# API Layer
cd SGKPortalApp.ApiLayer
dotnet run

# Presentation Layer
cd SGKPortalApp.PresentationLayer
dotnet run
```

6. **Access Application**
```
API: https://localhost:7001
UI: https://localhost:7002
Swagger: https://localhost:7001/swagger
```

## 📝 API Endpoints Overview

### Authentication
- POST `/api/auth/login` - Kullanıcı girişi
- POST `/api/auth/register` - Yeni kullanıcı kaydı
- POST `/api/auth/refresh-token` - Token yenileme
- POST `/api/auth/logout` - Çıkış

### Personel
- GET `/api/personel` - Personel listesi
- GET `/api/personel/{id}` - Personel detay
- POST `/api/personel` - Personel oluştur
- PUT `/api/personel/{id}` - Personel güncelle
- DELETE `/api/personel/{id}` - Personel sil
- GET `/api/personel/{id}/cocuklar` - Personel çocukları

### Sıramatik
- GET `/api/banko` - Banko listesi
- GET `/api/banko/{id}/durum` - Banko durum
- POST `/api/sira` - Yeni sıra oluştur
- PUT `/api/sira/{id}/cagirildi` - Sıra çağır
- GET `/api/sira/istatistik` - Sıra istatistikleri

### PDKS
- GET `/api/pdks/giris-cikis` - Giriş/Çıkış listesi
- POST `/api/pdks/izin-talep` - İzin talebi oluştur
- PUT `/api/pdks/izin-onay/{id}` - İzin onayla
- GET `/api/pdks/mesai-rapor` - Mesai raporu
- GET `/api/pdks/izin-bakiye/{personelId}` - İzin bakiyesi

### Eshot
- GET `/api/eshot/hareket` - Hareket listesi
- POST `/api/eshot/bakiye-yukle` - Bakiye yükle

-- Personel tablosu
CREATE INDEX IX_Personel_TcKimlikNo ON PER_Personeller(TcKimlikNo);

## 🔐 Security Best Practices

1. **Password Policy**
   - Minimum 8 karakter
   - En az 1 büyük harf
   - En az 1 küçük harf
   - En az 1 rakam
   - En az 1 özel karakter

2. **JWT Token**
   - Access Token: 60 dakika
   - Refresh Token: 7 gün
   - Token rotation aktif

3. **API Rate Limiting**
   - 100 request / dakika / IP
   - 1000 request / saat / IP

4. **CORS Policy**
   - Sadece belirlenen origin'lere izin
   - Credentials: true
